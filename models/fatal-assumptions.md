# 致命假设验证

识别点子背后最核心的假设，然后判断这些假设是否站得住脚。
这是整个评估流程中最关键的一步——如果核心假设是错的，点子就不可行。

## 核心思想

每个创业点子都建立在几个假设之上。有些假设即使错了也不致命，
但**致命假设**错了，整个点子就崩塌。

例子：
- 点子："做一个 AI 驱动的简历优化工具"
- 致命假设 1："求职者愿意为简历优化付费"
- 致命假设 2："AI 生成的简历比用户自己写的有明显优势"
- 如果假设 1 不成立 → 没有收入 → 点子失败
- 如果假设 2 不成立 → 没有价值 → 点子失败

## 执行步骤

### 第一步：列出所有假设
从用户的点子描述中，识别隐含的假设。通常包含：

| 假设类型 | 问题 | 示例 |
|----------|------|------|
| 需求假设 | 真的有人需要这个吗？ | "求职者需要简历优化" |
| 付费假设 | 用户愿意付钱吗？ | "求职者愿意花 50 元优化简历" |
| 技术假设 | 技术上能实现吗？ | "AI 能生成高质量简历" |
| 获客假设 | 能找到用户吗？ | "能在 LinkedIn 上低成本获客" |
| 规模假设 | 市场够大吗？ | "每月有足够多的求职者" |

### 第二步：找出致命假设
不是所有假设都同等重要。用这个方法筛选：

1. 问自己："如果这个假设是错的，点子还能成立吗？"
2. 如果答案是"不能"→ 这是致命假设
3. 一个点子通常有 1-3 个致命假设

### 第三步：验证致命假设
对每个致命假设，评估可验证性：

| 验证方法 | 成本 | 时间 | 可靠度 |
|----------|------|------|--------|
| 搜索已有数据和报告 | 低 | 快 | 中 |
| 看竞品的用户量和评价 | 低 | 快 | 中 |
| 发帖/发问卷测试反应 | 低 | 中 | 低 |
| 做 landing page 看注册量 | 中 | 中 | 高 |
| 手动提供该服务给 10 个人 | 中 | 慢 | 高 |

对于 OPC 来说，优先用低成本方法快速验证。团队同理，但可以利用多人并行验证。

## 判定规则

```
致命假设 < 2 个且都可低成本验证 → 🟢 风险可控
致命假设 2-3 个但至少有 1 个无法验证 → 🟡 需要先验证再决定
致命假设 > 3 个 → 🔴 假设太多，不确定性太高
发现致命假设明显不成立 → 🔴 直接放弃
```

## OPC 视角适配

OPC 的最大优势是可以快速验证。如果点子有致命假设但你能在 1 周内验证，
那不是风险，那是机会。真正的风险是**那些你验证不了或验证成本很高的假设**。

OPC 应该害怕的致命假设：
- "等做出来用户就会来" — 不会的，获客是最难的部分
- "这个市场应该挺大的" — "应该"不是数据
- "技术上应该没问题" — "应该"不是验证

## 团队视角适配

团队验证假设的方式和 OPC 不同，资源更多但速度可能更慢：

- **验证速度**：团队可以并行验证多个假设（不同人负责不同验证实验），
  但团队决策通常比 OPC 慢——需要讨论、对齐、分工。要警惕"验证 paralysis"。

- **致命假设的数量容忍度**：OPC 超过 3 个致命假设就应该放弃，团队可以容忍
  更多（比如 4-5 个），因为团队能分配资源同时验证。但如果超过 5 个，
  仍然说明不确定性太高。

- **团队特有的致命假设**：
  - "团队能保持稳定" — 核心成员离职会让项目瘫痪
  - "团队对方向的认同是一致的" — 联合创始人分歧是创业失败的常见原因
  - "团队有足够的资金支撑到验证完成" — 资金链断裂比假设不成立更致命

- **验证成本的计算方式不同**：OPC 的验证成本主要是时间，团队的验证成本
  还包括工资、办公等固定支出。验证周期越长，成本越高。

- ...

### 验证建议
