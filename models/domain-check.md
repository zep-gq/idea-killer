# 领域专项检查（Domain-Specific Check）

根据点子所属的行业领域，匹配特定的检查维度——合规风险、行业特殊风险、
关键成功因素、常见失败模式。这是通用模型，不是只服务某个领域。

## 核心思想

不同行业有不同的"潜规则"和"隐形杀手"。一个社交产品最大的风险是内容审核，
一个金融产品的最大风险是牌照合规，一个教育产品的最大风险是未成年人保护。
通用的评估模型（五力分析、精益画布等）看不到这些行业特有的坑，
领域专项检查就是为了补这个盲区。

## 第一步：识别领域

从用户的点子描述中，判断涉及哪些领域。一个点子可能涉及多个领域。

### 领域识别关键词映射

| 领域 | 关键词信号 |
|------|-----------|
| 🏘️ 社交/社区 | 社交、社区、论坛、聊天、匹配、交友、匿名、分享、UGC、粉丝、圈层 |
| 💰 金融/支付 | 支付、借贷、理财、保险、交易、投资、众筹、虚拟货币、钱包、征信 |
| 🏥 医疗/健康 | 医疗、诊断、药品、心理咨询、健康数据、诊疗、处方、问诊 |
| 📚 教育/青少年 | 课程、培训、学习、考试、辅导、青少年、K12、学生、校园 |
| 🛒 电商/零售 | 交易、买卖、商城、供应链、物流、仓储、商品、店铺 |
| 📰 内容/媒体 | 新闻、资讯、视频、音频、直播、出版、自媒体、版权 |
| 💼 企业服务/SaaS | 企业、B2B、办公、协作、CRM、ERP、HR、流程 |
| 🤖 AI/数据 | AI、模型、算法、数据、训练、推理、自动化 |

## 第二步：领域专项评估

对每个匹配的领域，执行以下 4 维检查。下面的内容是每个领域的
详细检查清单——只读你匹配到的领域部分，跳过其余的。

---

### 🏘️ 社交/社区

#### 合规要求
- **内容审核**：UGC 内容在中国需要遵守《网络安全法》《互联网信息服务管理办法》，
  要求"后台实名、前台自愿"。需要建立内容审核机制（关键词过滤 + 人工审核 + AI 审核）
- **未成年人保护**：如果用户可能包含未成年人，需遵守《未成年人网络保护条例》，
  需要青少年模式、时间限制、内容过滤
- **数据隐私**：社交产品收集大量用户数据，需遵守《个人信息保护法》，
  欧盟用户需 GDPR 合规
- **社区备案**：社区/论坛类产品可能需要 ICP 备案 + 公安备案

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 冷启动死亡谷 | 没有用户就没有内容，没有内容就没有用户，社交产品最常见的死法 | 🔴 |
| 网络暴力/极端内容 | 一条极端内容可能导致产品下架（Yik Yak、Sarahah 死因） | 🔴 |
| 内容审核成本失控 | 违规内容比例随用户量线性甚至指数增长，审核成本同步增长 | 🔴 |
| 社区氛围恶化 | 早期用户与晚期用户不兼容，社区调性随扩张稀释 | 🟡 |
| 留存困难 | 匿名/弱关系社交缺乏关系链锁定，用完即走 | 🟡 |
| 变现困难 | 匿名用户无法画像，广告价值低；用户不愿为社交功能付费 | 🔴 |

#### 关键成功因素
1. 找到冷启动的种子用户群（小而聚焦，而非大而泛）
2. 设计自转的内容飞轮（用户发帖 → 其他用户互动 → 激励发帖者继续发）
3. 第一天就有内容审核方案，而不是出事了再补
4. 寄生大平台冷启动（NGL 借 Instagram、Sendit 借 Snapchat），而不是从零建社交网络

#### 常见失败模式
- Secret：匿名+社交图谱 = 熟人恶意八卦
- Yik Yak：校园霸凌失控 → 被迫去匿名 → 用户流失
- Sarahah：无审核机制 → 骚扰泛滥 → 应用商店下架

### 💰 金融/支付

#### 合规要求
- **牌照门槛**：支付业务需要支付牌照，借贷需要小贷/消金牌照，
  理财需要基金销售/投顾牌照。没有牌照做金融 = 非法经营
- **反洗钱（AML/KYC）**：需要实名认证、交易监控、可疑交易报告
- **数据安全**：金融数据属于敏感个人信息，合规要求极高
- **利率限制**：借贷利率有司法保护上限（中国目前 LPR 4 倍）

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 无牌经营 | 没有牌照做金融业务是违法的，不是"先做了再补"的问题 | 🔴 |
| 欺诈/坏账 | 金融欺诈损失是直接的财务损失，不是 PR 问题 | 🔴 |
| 监管政策突变 | 金融监管政策变化频繁，一个文件就能让业务模式不合法 | 🔴 |
| 资金安全 | 用户资金安全问题可能导致刑事责任 | 🔴 |
| 信用风险 | 经济下行周期坏账率上升 | 🟡 |

#### 关键成功因素
1. 先确认需要什么牌照，再设计产品
2. 风控模型是核心——没有风控能力不要碰金融
3. 合规成本要算进 MVP 预算，不是"以后再说"

#### 常见失败模式
- 大量 P2P 平台因监管收紧集体覆灭
- 没有风控能力的团队做借贷，坏账率失控

---

### 🏥 医疗/健康

#### 合规要求
- **医疗资质**：诊断、处方类功能需要《医疗机构执业许可证》，
  提供医疗建议的人需要有执业资格
- **药品管理**：涉及药品销售/推荐需要《药品经营许可证》
- **健康数据**：健康信息属于敏感个人信息，数据处理合规要求高
- **广告限制**：医疗广告有严格审查制度，很多宣传话术不合规

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 误诊/延误治疗 | 提供医疗建议如果导致用户延误治疗，可能承担法律责任 | 🔴 |
| 资质缺失 | 没有医疗资质做诊断/治疗建议是非法行医 | 🔴 |
| 数据泄露 | 健康数据泄露的后果比普通数据严重得多 | 🔴 |
| 责任边界模糊 | "我只是提供信息不是诊断"的辩护在法庭上经常站不住脚 | 🟡 |

#### 关键成功因素
1. 明确产品是"健康信息工具"而非"医疗诊断工具"，避免踩线
2. 如果涉及医疗行为，与有资质的医疗机构合作
3. 免责声明和用户知情同意必须做足

---

### 📚 教育/青少年

#### 合规要求
- **双减政策（中国）**：K12 学科类培训受到严格限制，政策风险极高
- **未成年人保护**：需要青少年模式、时间限制、付费限制、内容审核
- **教育资质**：在线教育平台可能需要教育类许可证
- **广告限制**：面向未成年人的广告有严格限制
- **COPPA（海外）**：如果面向 13 岁以下用户，需要遵守 COPPA

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 政策风险 | 教育政策变化频繁且影响巨大（双减一夜改变整个行业） | 🔴 |
| 家长/学校抵触 | 如果产品不受家长/学校认可，会被抵制甚至举报 | 🔴 |
| 青少年安全 | 任何涉及未成年人的安全事故都会放大 10 倍 | 🔴 |
| 季节性 | 教育产品有明显的寒暑假淡旺季 | 🟡 |

### 🛒 电商/零售

#### 合规要求
- **经营资质**：需要营业执照、食品/特殊商品需要额外许可
- **消费者保护**：7 天无理由退货、产品质量责任、假货责任
- **广告合规**：商品描述、价格标注、促销规则有法律要求
- **跨境**：涉及进出口需要海关、税务合规

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 供应链复杂度 | 仓储、物流、售后、退换货——每个环节都是坑 | 🔴 |
| 假货/质量纠纷 | 一件假货就可能摧毁平台信誉 | 🔴 |
| 现金流压力 | 先采购后销售，库存积压 = 现金流断裂 | 🔴 |
| 获客成本高 | 电商获客成本在所有行业中名列前茅 | 🟡 |

---

### 📰 内容/媒体

#### 合规要求
- **内容审核**：与社交/社区类似，UGC 内容需要审核机制
- **版权**：转载、引用、二次创作需要处理版权问题
- **新闻资质**：新闻/时事类内容需要《互联网新闻信息服务许可证》
- **算法推荐**：中国《互联网信息服务算法推荐管理规定》要求算法透明

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 版权纠纷 | 未经授权使用他人内容是直接的法律风险 | 🔴 |
| 内容合规红线 | 涉及时政、社会热点的内容触碰红线后果严重 | 🔴 |
| 内容质量滑坡 | 为了流量放低内容门槛 → 劣币驱逐良币 | 🟡 |

---

### 💼 企业服务/SaaS

#### 合规要求
- **数据安全**：企业客户数据安全要求高，需要 SOC 2 / ISO 27001 等认证（海外市场）
- **合同合规**：与企业的合同中数据归属、SLA、责任限制条款
- **行业监管**：服务特定行业（如金融、医疗）时需满足该行业的合规要求

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 销售周期长 | B2B 从接触到签约通常 3-12 个月，OPC 资金链压力 | 🔴 |
| 定制化需求 | 企业客户总要求定制，标准产品难以满足 | 🟡 |
| 客户集中风险 | 少数大客户贡献大部分收入，一个客户流失影响巨大 | 🟡 |
| 续费率 | SaaS 的生命线是续费率，低于 80% 就不健康 | 🟡 |

#### 关键成功因素
1. OPC 应该做自助式 SaaS（不需要销售团队），不做需要定制交付的企业软件
2. 产品驱动增长（PLG）比销售驱动更适合小团队
3. 先服务 SMB 而非大企业——客单价低但决策快、要求少

### 🤖 AI/数据

#### 合规要求
- **生成式 AI 备案**：中国《生成式人工智能服务管理暂行办法》要求大模型备案
- **数据来源合规**：训练数据的版权和隐私合规
- **深度合成标识**：AI 生成内容需要标识（深度合成管理规定）
- **算法备案**：具有舆论属性的算法需要备案
- **数据出境**：涉及跨境数据传输需要安全评估

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 模型成本 | API 调用成本随用户量线性增长，单位经济模型容易算不过来 | 🔴 |
| 幻觉/准确性 | AI 输出错误信息可能导致用户损失和法律责任 | 🔴 |
| 平台依赖 | 严重依赖 OpenAI/Claude 等大模型 API，定价和政策变化不可控 | 🔴 |
| 同质化 | 套壳 AI 产品壁垒极低，别人也能做 | 🟡 |

### 🍔 餐饮/食品

#### 合规要求
- **食品经营许可**：需要《食品经营许可证》
- **卫生标准**：食品安全法要求，卫生不达标直接关停
- **营养健康标识**：预包装食品需要营养标签

#### 行业特殊风险
| 风险 | 说明 | 严重程度 |
|------|------|----------|
| 食品安全事故 | 一起食品安全事件就可能终结品牌 | 🔴 |
| 利润率低 | 餐饮行业平均利润率 5-15%，抗风险能力弱 | 🟡 |
| 运营复杂度 | 食材采购、库存、人员排班——运营细节极多 | 🟡 |

### 🏠 房产/住宿

#### 合规要求
- **房地产经纪资质**：撮合房产交易需要资质
- **民宿/短租**：各地对短租的监管政策不同，部分城市严格限制
- **消防安全**：住宿类产品需要消防验收

### 🚗 出行/交通

- **网约车资质**：需要《网络预约出租汽车经营许可证》
- **交通运营许可**：涉及客运需要道路运输经营许可证

### 🎮 游戏/娱乐

#### 行业特殊风险

| 开发周期长 | 游戏开发通常需要 6-24 个月，验证周期极长 | 🔴 |
| 同质化竞争 | 手游市场竞争极其激烈，买量成本极高 | 🔴 |

## 第三步：OPC / 团队视角适配

### OPC 专项提醒
- **合规成本是沉没成本**：很多合规要求（备案、审核系统）是固定成本，
  不管你有 10 个用户还是 10000 个用户都得投入。对 OPC 来说这意味着
  高固定成本 + 低规模效应 = 很难盈利
- **法律风险个人承担**：OPC 没有公司法人的保护伞，合规问题直接牵连个人
- **判断标准**：如果领域检查中发现 2 个以上 🔴 级别的合规要求，
  对 OPC 来说就是"不该碰的领域"

### 团队专项提醒
- **合规投入需要算进初始预算**：不是"有了收入再补合规"，
  而是合规是上线的必要条件
- **团队能承担更高的合规成本**：可以请法务、买保险、建立审核团队
- **但合规复杂度会随规模非线性增长**：100 个用户的审核和 10000 个用户
  的审核不是一个量级

## 判定规则

```
领域中有 🔴 级合规要求 + OPC 视角 → 强烈建议放弃（除非你有相关资质/经验）
领域中有 🔴 级合规要求 + 团队视角 → 需要将合规成本算进 MVP 预算
领域中有 3+ 个 🔴 级风险 → 无论什么视角都需要认真考虑是否值得进入
未匹配到任何领域 → 跳过本模型
```
