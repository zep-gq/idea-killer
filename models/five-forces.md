# 五力分析（OPC / 团队适配版）

用波特五力模型的思路分析市场环境，判断这个赛道对你是否友好。
经典五力模型面向大企业，这里做了 OPC 和团队两种视角的适配。

## 核心问题

从五个维度评估市场环境对"你做这个点子"是否有利：

### 1. 现有竞争者威胁
这个领域已有的竞争者有多强？

- **有利** 🟢：市场分散、无头部玩家、大公司看不上的小众市场
- **一般** 🟡：有一些竞品但没有绝对主导、利基市场有机会
- **不利** 🔴：巨头主导、头部集中、价格战激烈

问自己：**你能说出 3 个以上的直接竞品吗？如果能，为什么用户选你？**

### 2. 新进入者威胁
别人模仿你这个点子的门槛有多高？

- **有利** 🟢：有网络效应、数据壁垒、社区粘性
- **一般** 🟡：有一定的技术/内容壁垒但可被复制
- **不利** 🔴：几乎无壁垒、代码/模式容易被抄、纯靠执行力

问自己：**如果别人看到你做成功了，他们多久能抄出来？**

### 3. 替代品威胁
用户有没有其他方式解决同样的问题？

- **有利** 🟢：现有替代方案很差、用户在忍受痛苦
- **一般** 🟡：有替代方案但不够好、切换成本高
- **不利** 🔴：免费替代品多、用户手动也能搞定、AI 正在消灭这个需求

问自己：**如果这个产品不存在，用户现在怎么解决这个问题？**

### 4. 买方议价能力
目标用户有议价能力吗？

- **有利** 🟢：用户分散、付费能力强、对价格不敏感
- **一般** 🟡：用户有一定选择但愿意为好产品付费
- **不利** 🔴：用户集中在少数大客户、对价格极度敏感、免费预期强

问自己：**你的目标用户是 B2B（企业付费）还是 B2C（个人付费）？**

### 5. 供应商议价能力
你依赖的外部力量有多强？

- **有利** 🟢：技术/平台依赖少、供应链简单
- **一般** 🟡：依赖一两个平台但可替代
- **不利** 🔴：严重依赖某个平台/API/渠道、对方随时可能改变规则

问自己：**如果 OpenAI/Apple/Google 改变政策，你的产品还能活吗？**

## 判定规则

```
3+ 个维度有利 → 市场环境友好，值得进入
2 个不利 + 无有利 → 市场环境恶劣，建议换方向
供应商议价不利 + 新进入者威胁不利 → 死亡组合（做了也被抄/被卡脖子）
其余情况 → 需要更深入分析，进入致命假设验证
```

## OPC 视角适配

OPC 应该特别关注两件事：

1. **供应商依赖**：OPC 没有资源应对平台政策变化。如果一个点子严重依赖某个
   API 或平台（比如依赖 OpenAI API、依赖 App Store 分发、依赖某个社交平台），
   这就是一个结构性风险。

2. **新进入者威胁**：OPC 的护城河通常不是技术壁垒，而是速度、内容、社区、
   或领域知识。如果点子的唯一优势是"我想到了"，这不是优势——别人也会想到。

## 团队视角适配

团队的评估重点和 OPC 有所不同：

1. **供应商依赖**：团队有更多资源应对平台变化（可以快速迁移、多平台布局），
   但依赖的代价也更高——团队一旦投入了大量人力在某个平台上，迁移成本远高于 OPC。
   评估时要问："如果平台政策变了，团队需要多长时间迁移？这期间工资怎么发？"

2. **新进入者威胁**：团队可以建立技术壁垒（更复杂的系统、更好的性能），
   但要警惕"技术壁垒幻觉"——大部分初创团队的技术壁垒并没有想象中那么高。
   真正的壁垒通常来自数据积累、用户网络效应或领域深度。

3. **现有竞争者**：团队可以正面对抗更强的竞品，但需要明确差异化方向。
   不要和巨头比功能全面，要在某个垂直点上做到极致。

4. **买方议价**：团队可以服务 B 端大客户（OPC 通常做不了），但也意味着
   收入可能集中在少数客户上，客户流失的风险更大。
