# 案例研究 Agent

## 何时使用
仅在 Deep 模式下使用。Quick 模式跳过此文件。

## 执行方式
使用 WebSearch 工具执行搜索。共 2 组搜索，分别研究成功和失败案例。
搜索完成后，提取可复用的经验教训。

## 搜索策略

### 地区适配

搜索前先判断点子的目标市场：
- **面向中国用户**：增加中文搜索 + 国内平台搜索
- **面向海外用户**：使用英文搜索策略
- **用户没说 / 不确定**：中英文都搜，覆盖更全面

### 第一组：成功案例
搜索类似赛道中成功的创业案例，了解他们做对了什么。

搜索 query 示例（根据用户的点子替换关键词，按地区选择）：

**通用搜索：**
- `"{领域} 创业成功案例 独立开发者"`
- `"{domain} indie hacker success story solo founder"`

**海外侧重：**
- `"site:indiehackers.com {related keywords}"`
- `"{domain} solo founder MRR revenue"`

**中国侧重：**
- `"{领域} 一人公司 年收入 案例"`
- `"{领域} 独立开发者 月入 {近2-3年}"`
- `"site:即刻.com {related keywords}"`

关注信息：
- 创始人背景（是否 OPC）
- 从开始到盈利用了多久
- 关键转折点是什么
- 早期获客方式
- 当前收入规模（MRR/ARR）

### 第二组：失败案例
搜索类似赛道中失败的案例，了解他们踩了什么坑。

搜索 query 示例：
- `"{领域} 创业失败 关门 倒闭 原因"`
- `"{domain} startup failed shutdown postmortem"`
- `"why {type of product} failed"`
- `"site:failory.com {related keywords}"`

关注信息：
- 失败的根本原因（不是表面原因）
- 烧了多久的钱/时间
- 失败前有没有预警信号
- 如果重来，创始人会怎么做
