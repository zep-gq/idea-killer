# MVP 理论研究 Agent

## 何时使用
仅在 Deep 模式下使用。Quick 模式跳过此文件。

## 执行方式
使用 WebSearch 工具执行搜索。共 2 组搜索，分别寻找该领域的 MVP 最佳实践
和验证方法论。搜索完成后，提炼出可操作的建议。

## 搜索策略

### 第一组：领域 MVP 实践
搜索该特定领域的 MVP 构建经验和最佳实践。

搜索 query 示例（根据用户的点子替换关键词）：
- `"{领域} MVP 最佳实践 最小可行产品"`
- `"{domain} MVP best practices minimum viable product"`
- `"{领域} 如何验证想法 landing page 预售"`
- `"how to validate {type of product} idea before building"`

关注信息：
- 该领域常见的 MVP 形态（landing page、手动服务、demo）
- 从 MVP 到 PMF 通常需要多久
- 哪些功能是 MVP 必须有的，哪些可以后续加
- 该领域用户的付费意愿验证方法

### 第二组：快速验证方法论
搜索通用的快速验证方法论，特别是适合 OPC 和小团队的方法。

搜索 query 示例：
- `"Pieter Levels 验证想法 方法 快速"`
- `"solo founder idea validation framework"`
- `"OPC 一人公司 验证点子 方法 {近2-3年}"`
- `"lean startup validation experiment checklist"`

关注信息：
- 验证一个点子需要多长时间
- 低成本验证的具体方法（预售、候补名单、手动服务）
- 什么时候该放弃（pivot 或 kill 的判断标准）
- 有没有现成的验证清单或框架
