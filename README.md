# Idea Killer — 点子杀手 🔪💡

帮助 OPC（一人公司）或小团队在投入时间之前，用真实数据和结构化框架快速评估和淘汰创业点子。

**核心信念：90%+ 的点子应该在前 5 分钟被淘汰。杀得快比选得准更重要。**

## 特性

- **双模式**：Quick（5 分钟快速判断）+ Deep（完整调研 + 结构化评估 + MVP 文档）
- **双视角**：OPC（一人公司）/ 团队，评估标准自动适配
- **8 个评估模型**：红黄绿灯、领域专项检查、价值-复杂度矩阵、五力分析、致命假设验证、精益画布、RICE 优先级、Levels 快速验证方法
- **3 个研究 Agent**（Deep 模式）：竞品研究、案例研究、MVP 理论
- **12 个行业专项检查**：社交、金融、医疗、教育、电商、内容、SaaS、AI、餐饮、房产、出行、游戏
- **地区适配**：根据目标市场自动选择中/英文搜索策略
- **数据可靠性保障**：搜索失败时自动标注，不会用无数据支撑的结论误导用户

## 使用方式

这是一个 [Claude Code](https://claude.com/claude-code) Skill，安装后可在对话中触发。

### 触发方式

当你在对话中提到以下类似内容时自动触发：

- "帮我评估一个点子"
- "这个想法靠谱吗"
- "我想做一个..."
- "值得做吗"
- "帮我看看这个方向"

### Quick 模式示例

```
"帮我快速看看，我想做一个程序员副业接单平台"
```

5 分钟内给出红黄绿灯评估 + 致命假设检查 + Levels 快速验证。

### Deep 模式示例

```
"帮我认真评估一下，我想做一个 AI 心理咨询聊天机器人，一个人做，周末项目"
```

并行启动 3 个研究 Agent 搜索竞品/案例/理论，然后依次走 8 个评估模型，输出完整报告和 MVP 文档。

## 铁律

```
没有调研就没有发言权 — 每个结论必须有搜索到的数据支撑
杀得快比选得准更重要 — 遇到红灯就停，不要试图拯救一个坏点子
一个周末能上线吗？ — OPC 标准；团队放宽到两周
跟随用户语言 — 用户用什么语言描述点子，就用什么语言输出评估
淘汰不等于否定 — 即使建议放弃，也要肯定闪光点，提供发散思路和改进方向
```
