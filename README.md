# 🔪 Idea Killer

### _杀得快，比选得准更重要_

**用数据杀掉坏点子，把时间留给好点子。** 🔪💡

## ✨ 核心信念

> **90%+ 的点子应该在前 5 分钟被淘汰。**
> 一个周末做不出来的、一个人扛不住的、没有真实数据支撑的——都是坏点子。
> 杀得快比选得准更重要。淘汰不等于否定，但犹豫等于浪费时间。

## 🚀 安装

### 方式一：一行命令

```bash
npx skills add zep-gq/idea-killer
```

支持 55+ AI agent runtime（Claude Code、Codex、Cursor、Gemini CLI 等）。需要指定 runtime 时加 `-a` 参数：

```bash
npx skills add zep-gq/idea-killer -a claude-code
```

### 方式二：手动安装

```bash
git clone https://github.com/zep-gq/idea-killer.git
```

将 `idea-killer` 目录放入你的 agent 的 skills 目录即可。常见路径：

| Runtime | Skills 目录 |
|---------|-------------|
| Claude Code | `~/.claude/skills/` |
| Cursor | `.cursor/skills/` |
| Codex | `.codex/skills/` |
| 其他 | 参考对应文档 |

### 方式三：直接使用

即使你的 runtime 不支持 skills 协议，也可以直接把 `SKILL.md` 的内容粘贴进对话——它本质就是一份 markdown 文档。

## 📋 真实案例

> 以下是使用 Idea Killer 实际评估的真实案例。结论均基于搜索到的市场数据。

---

### 🔴 案例 1：匿名社交树洞 — Deep 模式

**点子**：一个让任何人都可以匿名倾诉情绪、分享秘密的泛人群平台。OPC 一个人做，周末项目。

**结论：🔴 建议放弃**

| 评估模型 | 结论 | 关键发现 |
|----------|------|----------|
| 🚦 红黄绿灯 | 🔴 建议放弃 | 3 红灯（差异化、付费意愿、OPC 可行性），0 绿灯 |
| 🏗️ 领域专项检查 | 🔴 高风险 | 社交领域：内容审核成本失控 + 网络暴力 + 合规红线 |
| 📊 价值-复杂度 | 🔴 没意义 | 低价值 × 高复杂度，最差象限 |
| ⚔️ 五力分析 | 🔴 恶劣 | 5 个维度全部不利 |
| 💀 致命假设 | 🔴 未通过 | 5 个致命假设中 3 个大概率不成立 |
| 📋~⚡ | ⏭️ 跳过 | 提前终止：结论已明确 |

**Agent 研究发现**：
- 赛道死亡率 ~90%：Secret、Yik Yak、Sarahah、Formspring、无秘全部关闭
- 失败根因 100% 与内容审核/安全管理失控相关
- 成功案例全部依赖寄生大平台冷启动（NGL 借 Instagram、Sendit 借 Snapchat）

**闪光点 & 发散**：

> 💡 "情绪需求"方向是对的，但载体错了

| 替代方向 | 说明 | 可行性 |
|----------|------|--------|
| 🅰️ 寄生大平台做匿名互动小程序 | 不做独立 App，做微信小程序，审核压力交给平台 | ⭐⭐⭐ |
| 🅱️ 垂直人群匿名倾诉 | 聚焦程序员/考研等垂直场景，审核难度大幅降低 | ⭐⭐⭐ |
| 🅲️ **AI 情绪陪伴工具（强烈推荐）** | 去掉社交，用户倾诉 AI 回应。无需审核、无需冷启动、周末可做 | ⭐⭐⭐⭐⭐ |

> 🪞 _"用户真正想要的是'匿名的社交'，还是'不被评判的表达空间'？如果是后者，AI 陪伴可能比匿名社交更好地满足这个需求。"_

### 🟡 案例 2：AI 心理咨询聊天机器人 — Quick 模式

**点子**：焦虑或心情不好时可以跟 AI 聊天，用心理学方法疏导情绪。OPC 一个人做，周末项目。

**结论：🟡 需要验证**

| 维度 | 灯 | 判断 |
|------|----|------|
| 必要性 | 🟡 | 需求真实但替代方案多（Calm、Headspace、Character.ai） |
| 差异化 | 🟡 | 比冥想 App 互动性强、比真人咨询便宜，但 Woebot/Wysa 已在做 |
| 可触达 | 🟢 | "焦虑""情绪管理"搜索热词，内容获客路径清晰 |
| 付费意愿 | 🟡 | 赛道有付费先例（Calm $69.99/年），但"AI 聊天心理"付费习惯未验证 |
| OPC 可行性 | 🟡 | 技术✅ + 运营✅（1 对 1 对话无需审核）+ 合规⚠️（心理健康的灰色地带） |

**领域专项检查触发**：

```
匹配领域：🤖 AI/数据 + 🏥 医疗/健康

⚠️ 合规风险：
  • 生成式 AI 需要备案（《生成式人工智能服务管理暂行办法》）
  • 必须定位为"情绪支持工具"，不能触碰"诊断/治疗"红线
  • AI 输出如果导致用户延误就医，可能承担法律责任
```

**Levels 快速验证**：

| 问题 | 判定 |
|------|------|
| 一个周末能上线？ | ✅ 大模型 API + prompt 工程 |
| 能用 no-code？ | ✅ Coze/Dify 直接搭 |
| 有人为类似产品付费？ | ✅ Woebot、Wysa 已验证 |

**闪光点 & 发散**：

| 替代方向 | 说明 |
|----------|------|
| 🅰️ 垂直场景 AI 陪伴 | "考研焦虑陪伴""新手妈妈情绪支持"——定位越垂直，合规风险越低 |
| 🅱️ **AI 心理日记** | 用户写日记，AI 给心理学框架的反馈。产品更轻、合规风险更低 |
| 🅲️ 先出海做英文版 | 海外 FDA 对 wellness 类 App 监管宽松，先验证再回国 |

### 🟢 案例 3：Chrome 插件 — 帮跨境电商卖家自动比价 — Quick 模式

**点子**：一个 Chrome 插件，在卖家浏览 1688/淘宝时自动显示同款商品在 Amazon 的售价和利润空间。

**结论：🟢 值得尝试**

| 维度 | 灯 | 判断 |
|------|----|------|
| 必要性 | 🟢 | 跨境卖家核心痛点——选品时手动比价极其耗时 |
| 差异化 | 🟢 | 1688 → Amazon 自动比价，市面没有直接竞品 |
| 可触达 | 🟢 | 卖家聚集在 TikTok/微信社群/知乎，精准触达 |
| 付费意愿 | 🟢 | 卖家已经在为 ERP 工具付费（每月 $20-50），付费习惯成熟 |
| OPC 可行性 | 🟢 | Chrome 插件 + 简单 API，周末可做 |

**Levels 快速验证**：3/3 ✅ 通过

**建议**：立即做 MVP，先在一个跨境卖家微信群验证。

### ⚔️ 案例 4：大学生技能交换平台 — Deep 模式

**点子**：让大学生可以用自己的技能（PS、编程、英语）交换别人的技能，不涉及金钱。团队 3 人做。

**结论：🟡 建议修改后推进**

| 评估模型 | 结论 | 关键发现 |
|----------|------|----------|
| 🚦 红黄绿灯 | 🟡 需要验证 | 必要性🟢 + 差异化🟡 + 可触达🟢 + 付费意愿🔴 + 团队可行性🟢 |
| 🏗️ 领域专项检查 | 🟡 社交 + 教育 | 冷启动死亡谷 + 大学校园推广成本 |
| 📊 价值-复杂度 | 🟡 填充物 | 中价值 × 中复杂度 |
| ⚔️ 五力分析 | 🟡 一般 | 替代品多（社团、闲鱼、微信群），但免费技能交换无人做 |
| 💀 致命假设 | ⚠️ 不确定 | "大学生真的愿意用技能交换而非直接付费"——需要验证 |

**关键风险**：付费意愿红灯——纯免费模式没有收入，需要设计增值变现路径。

**建议修改**：不做"技能交换"，做"技能付费 + 首单免费试课"——保留社区感但有明确变现路径。

## 📐 评估流程

### Quick 模式

```
用户描述点子
    │
    ▼
🚦 红黄绿灯（5 个维度）
    │
    ├── 🔴 多红灯 → 直接建议放弃
    │
    ├── 🟡 领域风险扫描 → 是否匹配高风险行业？
    │
    └── 🟢 / 少量黄灯
         │
         ▼
    💀 致命假设快速检查
         │
         ▼
    ⚡ Levels 快速验证（3 问）
         │
         ▼
    📋 输出快速评估报告
```

### Deep 模式

```
用户描述点子
    │
    ▼
确认视角 + 提取核心信息
    │
    ├─────┬─────┐
    ▼     ▼     ▼
  🔎竞品 📖案例 🧪MVP    ← 3 个 Agent 并行搜索
    │     │     │
    └─────┴─────┘
          │
          ▼
    数据可靠性检查 ⚠️
          │
          ▼
  1. 🚦 红黄绿灯
  2. 🏗️ 领域专项检查    ← 新增：12 行业合规+风险
  3. 📊 价值-复杂度
  4. ⚔️ 五力分析
  5. 💀 致命假设        ← 可提前终止
  ─ ─ ─ ─ ─ ─ ─ ─
  6. 📋 精益画布
  7. 🎯 RICE 优先级
  8. ⚡ Levels 验证
          │
          ▼
    📄 终端概要 + MVP 文档
```

## ⚖️ 铁律

> 🔴 **没有调研就没有发言权** — 每个结论必须有搜索到的数据支撑
>
> ⚡ **杀得快比选得准更重要** — 遇到红灯就停，不要试图拯救一个坏点子
>
> ⏱️ **一个周末能上线吗？** — OPC 标准；团队放宽到两周
>
> 🌐 **跟随用户语言** — 用户用什么语言，就用什么语言输出
>
> 💡 **淘汰不等于否定** — 即使建议放弃，也要肯定闪光点，给出发散方向
